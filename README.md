# HANDMADE

핸드메이드 상품을 판매하고 구매할 수 있는 수제 쇼핑몰 웹 프로젝트입니다.
구매자는 상품 조회, 장바구니, 주문, 결제, 리뷰, 문의, 찜 기능을 사용할 수 있고, 판매자는 상품 등록, 상품 관리, 주문 관리, 교환/반품 처리를 할 수 있습니다.
또한 관리자는 판매자 승인, 상품 승인 상태, 신고 내역을 관리하여 쇼핑몰에 노출되는 상품과 이용자 활동을 관리할 수 있습니다.

## 프로젝트 소개

이 프로젝트는 핸드메이드 상품 판매 플랫폼을 주제로 제작한 Spring Boot 기반 쇼핑몰입니다.
일반 쇼핑몰의 핵심 흐름인 상품 조회 → 장바구니 → 주문/결제 → 배송 관리 → 리뷰/문의/신고 작성까지 구현했으며, 판매자와 관리자 기능도 함께 구성했습니다.

## 핵심 구현 내용

### 1. 상품 기능

상품은 승인 상태가 `APPROVED`인 경우에만 사용자 화면에 노출되도록 구성했습니다.
상품 등록 시 대표 이미지와 상세 이미지를 업로드할 수 있으며, 상품 설명에는 Toast UI Editor를 적용했습니다.

주요 구현 내용은 다음과 같습니다.

* 상품 등록
* 상품 수정
* 상품 숨김 처리
* 대표 이미지 업로드
* 상세 이미지 다중 업로드
* 카테고리별 상품 조회
* 상품 검색
* 할인 가격 계산
* 무료배송 조건 처리

### 2. 장바구니 기능

장바구니에서는 판매자별 상품 그룹을 구분하고, 선택한 상품만 주문할 수 있도록 구현했습니다.
수량 변경 시 상품별 합계와 전체 결제 예상 금액이 다시 계산되도록 구성했습니다.

주요 구현 내용은 다음과 같습니다.

* 장바구니 상품 추가
* 중복 상품 담기 시 수량 증가
* 상품 수량 변경
* 선택 상품 삭제
* 선택 상품 주문
* 배송비 계산
* 할인 금액 계산

### 3. 주문/결제 기능

장바구니 주문과 단건 주문을 분리해서 처리했습니다.
주문 생성 후 결제 페이지로 이동하며, 주문 상태에 따라 구매자 화면의 버튼이 다르게 출력되도록 구현했습니다.

주문 상태에 따라 다음과 같이 화면이 변경됩니다.

* 결제 전: 결제하기 / 주문취소
* 결제 완료: 결제취소
* 제작 준비중: 배송조회
* 배송중: 배송조회
* 배송완료: 리뷰쓰기 / 교환·반품 신청

### 4. 리뷰 기능

배송완료된 상품에 대해서만 리뷰를 작성할 수 있도록 구성했습니다.
리뷰에는 별점, 내용, 이미지를 등록할 수 있으며 상품 상세 페이지에서 리뷰를 확인할 수 있습니다.

주요 구현 내용은 다음과 같습니다.

* 작성 가능한 리뷰 조회
* 작성한 리뷰 조회
* 리뷰 이미지 등록
* 리뷰 이미지 모달
* 베스트 리뷰 정렬
* 이미지 많은 리뷰 우선 정렬

### 5. 문의 기능

사용자는 상품 상세 페이지에서 문의를 작성할 수 있고, 판매자는 판매자 페이지에서 문의를 확인하고 답변할 수 있도록 구성했습니다.

주요 구현 내용은 다음과 같습니다.

* 상품 문의 작성
* 상품별 문의 목록 조회
* 판매자 문의 관리
* 답변 등록
* 답변 완료 여부 표시

### 6. 찜 기능

상품 찜과 판매자 찜을 분리해서 구현했습니다.
사용자는 마이페이지에서 찜한 상품과 찜한 판매자를 한 번에 관리할 수 있습니다.

주요 구현 내용은 다음과 같습니다.

* 상품 찜 등록
* 상품 찜 취소
* 판매자 찜 등록
* 판매자 찜 취소
* 관심 목록 조회
* 찜 개수 표시

### 7. 판매자 주문 관리

판매자는 주문 상태를 변경할 수 있고, 구매자가 신청한 교환/반품 요청을 확인하고 처리할 수 있습니다.
주문 관리는 카드형/심플형 화면을 지원하도록 구성했습니다.

주요 구현 내용은 다음과 같습니다.

* 전체 주문 조회
* 주문 상태별 필터
* 주문 상태 변경
* 카드형 보기
* 심플형 보기
* 주문 상세 모달
* 교환/반품 요청 확인 및 반려
* 교환/반품 처리 완료

### 8. 관리자 기능

관리자는 쇼핑몰 운영에 필요한 승인, 상태 변경, 데이터 조회 기능을 처리할 수 있도록 구성했습니다.
대시보드에서는 승인 대기 판매자, 승인 대기 상품, 미처리 신고, 답변 대기 문의, 전체 회원 수와 같은 운영 현황을 확인할 수 있습니다.

판매자와 상품은 상태별로 조회하고 승인, 반려, 숨김 처리를 할 수 있으며, 처리 시 변경 전/후 상태와 메모를 관리 내역으로 남기도록 구성했습니다.
회원, 후기, 문의, 신고, 주문, 카테고리, 공지사항도 관리자 화면에서 조회하고 상태를 변경하거나 답변을 등록할 수 있습니다.

주요 구현 내용은 다음과 같습니다.

* 관리자 대시보드 현황 조회
* 판매자 상태별 조회 및 승인 / 반려 처리
* 상품 상태별 조회 및 승인 / 반려 / 숨김 처리
* 회원 목록 조회 및 계정 상태 변경
* 후기 목록 조회 및 노출 / 숨김 상태 변경
* 신고 목록 조회 및 처리 / 반려 상태 변경
* 문의 목록 조회, 답변 등록, 답변 상태 변경
* 주문 목록 조회 및 주문 상태 변경
* 카테고리 등록 / 수정 / 삭제
* 공지사항 목록 조회 및 상세 조회
* 관리자 처리 내역 조회
* 대상별 관리자 메모 등록 및 조회
* 검색, 상태, 기간, 정렬, 페이징 조건 처리

## 프로젝트에서 고민한 부분

### 주문 상태와 상품별 교환/반품 상태 분리

처음에는 `order_items`에 상태를 직접 넣어 주문 상태와 함께 관리하려 했지만, 주문 전체의 배송 상태와 상품별 교환/반품 상태는 역할이 다르다고 판단했습니다.
그래서 주문 배송 상태는 `orders.status`에서 관리하고, 교환/반품 요청은 `order_item_claims` 테이블로 분리했습니다.

이를 통해 주문은 배송 흐름을 담당하고, 상품별 교환/반품은 별도의 요청 데이터로 관리할 수 있게 되었습니다.

### 판매자 주문 관리 화면 구조

판매자 주문 관리는 단순 목록만 보여주는 방식보다 실제 쇼핑몰 관리자 페이지처럼 주문 상태, 상품 정보, 구매자 정보, 배송 정보, 교환/반품 상태를 한 화면에서 확인할 수 있도록 구성했습니다.

또한 카드형 보기와 심플형 보기를 나누어 판매자가 원하는 방식으로 주문을 확인할 수 있도록 했습니다.

### 이미지 처리

상품 대표 이미지는 `products.image_l`에 저장하고, 상세 이미지나 리뷰 이미지는 `images` 테이블에서 `target_type`, `target_id`로 구분해서 관리했습니다.
이를 통해 상품 이미지와 리뷰 이미지를 같은 이미지 테이블에서 재사용할 수 있도록 구성했습니다.

## 트러블슈팅

### MyBatis 컬럼명 오류

DB 컬럼과 Mapper에서 사용하는 컬럼명이 맞지 않아 `Unknown column` 오류가 발생했습니다.
해결을 위해 실제 DB 테이블 구조를 기준으로 Mapper SQL을 다시 확인하고, 존재하지 않는 컬럼 조건을 제거하거나 테이블 구조에 맞게 수정했습니다.

### JSP 이미지 경로 오류

상품 이미지가 깨지는 문제가 발생했습니다.
이미지 저장 경로와 화면 출력 경로가 다르게 처리되고 있어, 업로드 경로와 기본 이미지 경로를 분리해서 처리했습니다.

### 주문 상태 버튼 출력 문제

주문 상태에 따라 버튼이 정상적으로 출력되지 않는 문제가 있었습니다.
JSP에서 상태값 조건을 다시 정리하고, `PENDING`, `PAID`, `PREPARING`, `SHIPPING`, `DELIVERED`, `CANCELLED` 상태별로 버튼이 다르게 출력되도록 수정했습니다.

### 교환/반품 상태 처리 문제

교환/반품 요청 상태가 주문 상태와 섞이면서 화면 조건이 복잡해지는 문제가 있었습니다.
이를 해결하기 위해 `order_item_claims` 테이블을 따로 만들고, 상품별 교환/반품 상태를 별도로 조회해서 화면에 표시하도록 수정했습니다.

## 프로젝트 목적

이 프로젝트는 Spring Boot 기반 웹 개발 흐름을 익히고, 실제 쇼핑몰 서비스에서 사용되는 상품, 주문, 결제, 리뷰, 문의, 판매자 관리 기능을 직접 구현해보기 위해 제작했습니다.

단순 CRUD를 넘어서 사용자와 판매자의 흐름을 나누고, 주문 상태와 교환/반품 상태처럼 실제 서비스에서 발생할 수 있는 복잡한 상태 관리까지 구현하는 것을 목표로 했습니다.
